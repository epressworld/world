## Project Overview

This is the official website and documentation hub for epress, a decentralized Web3 blog and social network. It's a Next.js 16 application using React 19, JavaScript/JSX, and Tailwind CSS.

## Commands

```bash
npm run dev          # Start development server (localhost:3000)
npm run build        # Production build
npm start            # Start production server
npm run lint         # Run Biome linter
npm run lint:fix     # Run Biome with auto-fix
npm run docs:sync:epress  # Sync developer docs from ../epress repository
```

## Architecture

### Repository Relationship

This repository (`world`) is the documentation/website companion to the main `epress` node implementation. Developer API references are generated by syncing from `../epress` (sibling directory expected to exist locally).

### Directory Structure

- `src/app/` - Next.js App Router pages
  - `page.jsx` - Landing page
  - `whitepaper/page.jsx` - Renders WHITEPAPER.md with math/diagram support
  - `docs/` - Two-lane documentation: Node Operator docs and Developer docs (`developer/ewp/` for protocol, `developer/node/` for node implementation)
- `src/components/` - React components organized by feature (`home/`, `vision/`, `how-it-works/`, `developer/`, `shared/`)
- `src/content/developer/` - Generated JSON snapshots for developer docs (API endpoints, GraphQL operations, EIP-712 types)
- `src/lib/` - Utilities including `cn()` for Tailwind class merging
- `scripts/sync-epress-docs.mjs` - Extracts API/protocol info from `../epress` into JSON snapshots

### Key Patterns

- Path alias: `@/*` maps to `./src/*`
- Use `"use client"` directive for client-side components
- Theme handled by `next-themes` via `ThemeProvider` wrapper
- Tailwind custom classes: `btn-primary`, `btn-secondary`, `landing-card`, `concept-section-card`, etc.
- Color tokens: `dark-bg`, `dark-surface`, `dark-border`, `dark-text`, `dark-muted`, `primary` (orange #F7931A)

### Canonical Terminology

Use these exact terms:
- `epress world` - the decentralized social network
- `epress world protocol` - the interoperability protocol (EWP)
- `epress node` - the node software implementation

## Code Style

- JavaScript/JSX (not TypeScript)
- No comments unless explicitly requested
- Functional components with arrow functions
- Default exports for pages, named exports for utilities
- Component files: kebab-case (e.g., `concept-cards.jsx`)
- Run `npm run lint:fix` before committing

## Workflow Notes

- Main branch is production; deployed on Vercel at `epress.world`
- After syncing docs from epress, always run `npm run build` to verify
- All documentation and commit messages in English
